Hobbies

Freetime projects

  • Quadcopter building: starting from a hardware kit (frame, motors, propellers, and so on) and after assembling it (which required to fix various problem due to neglectful chineese providers) the project is to realise an autonomus fly controller, based on an AVR microcontroller, as a first step. It is "minimalistic", a kind of proof-of-concept that one can do basic autonomous flight, in 8-bit, with very few RAM space.
  • openPowerSwitch: the goal is to reproduce the features of an existing product I was using in Wyplay. a "power switch" is a multi-socket mains extension, which allows to control remotely and individually each outlet it holds through the network, using HTTP requests. The AVR microcontroller is wired to relays (one per outlet) to switch 220V AC, and to an ethernet/RJ45 interface for the network part. On the software side, I use uip (micro-IP), a tiny but full-featured IP stack written by Adam Dunkels. This project is also a chance (not done yet) to get into CAD software: freeCAD to design the casing (absolutely needed because of the 220VAC) and kicad, to maybe produce my own PCB design.
  • Automatic watering system: feeding water to potted vegetable and/or flowers. It implies sensors to detect the water needing, a pump, a tank, and so on. Nowadays I only played around with mouisture sensor, and found out they are not that accurate. I'm considering measuring total weight also (or instead), to deduced amout of evaporated water.
  • media "TV" box: integrating kodi on a raspberry pi, to be able to play various media, especially from the internet, from streaming sites (like Youtube), or IP TV channels (like ARTE+7)... I started this project because internet subscription is much more expensive in Belgium than in France. Indeed, I subscribed to a "naked" connection, without TV channels offer ; but I wanted to be able to browse contents, launch a medias or change the settings (like sound volume)... With a nice "10-feet interface", and without having to grab mouse and keyboard thanks to a dedicated Remote Control Unit.
  • Writing articles: on my technical blog, about various tests, tries, and small tinkering I do.
